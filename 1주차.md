
### 1. 학습 과정과 현재 자신이 생각하는 역량은 무엇인가요? (1000자 이내)
제가 생각하는 학습 과정의 핵심은 ‘지금 당장 필요하지 않더라도 다양한 기술과 사례를 경험해보는 것’입니다. 개발 환경은 계속 변화하고 새로운 기술도 빠르게 등장하기 때문에, 저는 폭넓은 경험을 통해 상황에 맞는 최적의 선택을 할 수 있는 개발자가 되고자 노력해왔습니다. 그래서 새로운 기술을 배우면 직접 실험해보고 프로젝트에 적용해보며, 제 도메인에 맞게 어떻게 사용할 수 있을지 항상 고민합니다.

이러한 저만의 학습 방식이 가장 잘 드러났던 경험은 첫 회사에서의 프로젝트입니다. 당시 저는 사수가 없는 환경에서 백엔드 시스템을 처음부터 끝까지 독립적으로 개발해야 했습니다. 설계부터 DB 모델링, 배포까지 모든 결정을 스스로 내려야 했고, 참고할 사람이 없었습니다. 그래서 저는 한 가지 원칙을 세웠습니다.

“대기업의 검증된 기술 사례를 내 환경과 비교·분석하여, 적용 가능한 부분만 선별하자.”

이 원칙을 바탕으로 여러 대기업의 기술 컨퍼런스 발표와 문서를 찾아 분석했습니다.
“이 회사는 어떤 이유로 이 아키텍처를 선택했지?”,
“내 프로젝트 규모에서도 이 방식이 유효할까?”
와 같은 질문을 스스로 던지며, 그대로 따라가는 것이 아니라 ‘제 환경에 맞게 재해석하는 방식’으로 학습했습니다.

이러한 경험을 바탕으로 저는 문제를 스스로 정의하고 해결 전략을 설계할 수 있으며, 대기업의 검증된 기술·아키텍처를 분석한 뒤 현재 조직의 리소스·트래픽·제약 조건에 맞게 재해석하여 최적의 방식으로 적용하는 ‘맥락 기반 설계 능력’을 핵심 역량으로 생각합니다. 단순히 기술을 그대로 사용하는 것이 아니라, 조직의 현실적 상황을 고려해 가장 현실적이고 효과적인 해결안을 선택하고 조정할 수 있는 유연함이 제 강점입니다.

이렇게 쌓은 경험의 조각들은 실제로 코인거래소 스테이킹 서비스에서 이자 지급 기능을 개발하며 Spring Batch를 적용할 때 큰 도움이 되었습니다. 초기 실험을 통해 배치 처리 방식과 트랜잭션 관리 방법을 익혀두었기 때문에, 실제 구현 과정에서도 문제 없이 빠르게 적용할 수 있었습니다.

그 결과, 스테이킹 서비스 신청부터 지급·해제까지 전 과정에서 이슈 티켓 0건을 달성했고, 서비스는 가상 환경에서 인기 상품으로 선정되는 성과를 얻었습니다.

### 2. 선택한 직군을 목표로 하게 된 이유는 무엇인가요? (1000자 이내)
반복적인 일을 자동화해 더 가치 있는 일에 집중할 수 있게 만드는 것에 큰 흥미가 있었습니다. 물론 자동화는 다양한 직군에서도 다룰 수 있지만, 저는 스마트폰이 급격하게 보급되던 시기에 “사람들의 일상과 산업 전체를 바꾸는 힘이 소프트웨어에 있다”는 사실을 직접 체감했습니다. 단순히 ‘작업을 줄이는 도구’가 아니라 세상의 기본 구조 자체를 바꾸는 역할이 개발자에 의해 만들어진다는 점이 매력적으로 다가왔습니다.

개발을 처음 배우며 작은 기능 하나가 실제 불편을 즉시 줄여주는 경험을 했고, 이때부터 “문제를 가장 근본적으로 해결할 수 있는 직업은 개발자”라는 확신을 갖게 되었습니다. 컴퓨터공학 전공을 통해 자료구조, 네트워크, OS 등 기반 지식을 배우면서 기술이 어떻게 세상을 움직이는지 구조적으로 이해할 수 있었고, 이를 검증하기 위해 다양한 프로젝트를 직접 만들며 실제 사용자 불편을 해결해보았습니다.

회사에 입사한 이후에는 이 확신이 더 분명해졌습니다. 서비스의 병목지점을 개선하거나 반복적인 운영 업무를 자동화했을 때 팀 전체의 효율이 크게 오르는 것을 경험했고, 특히 문제를 정확히 정의하고 해결하는 과정 자체가 개발자만이 할 수 있는 본질적인 일임을 체감했습니다. 최근에는 채팅 기능의 메시지 지연·전송 실패 문제를 개선해 고객센터 문의가 실제로 줄어드는 변화도 경험했습니다. 제가 만든 개선이 곧바로 사용자 경험 향상으로 이어지는 순간들이 개발자로서 큰 보람이 되었습니다.

저에게 개발은 기능을 만드는 일이 아니라 문제를 발견하고 기술적으로 해결하는 과정 그 자체입니다. 더 많은 사람의 시간을 절약하고, 불편을 줄이며, 기술로 세상을 더 효율적으로 만드는 데 기여하고 싶습니다. 이런 이유로 저는 개발자를 선택했고, 앞으로도 이 방향으로 성장하고 싶습니다.

### 3. 마음이 끌리는 무언가에 몰입하여 일정 수준의 결과물을 만들어 낸 경험이 있나요? (1000자 이내)
평소에 “언젠가는 세계적인 오픈소스에 기여해보고 싶다”는 목표가 있었습니다. 하지만 방대한 코드베이스를 어디서부터 파고들어야 할지 막막했고, 오픈소스 기여는 나와는 먼 이야기라고 느꼈습니다. 그러던 중 AI 코드 분석 도구(ChatGPT 등)를 활용하면서 상황이 크게 달라졌습니다. 복잡한 구조와 동작 의도를 빠르게 파악할 수 있게 되었고, “이제는 실행할 수 있겠다”는 확신이 생겼습니다.

그때 제가 선택한 프로젝트는 Spring Kafka였습니다. 실제 업무에서 Kafka를 사용하면서 특정 설정 객체의 toString() 메서드가 문자열을 + 연산으로 계속 이어 붙이는 방식으로 작성돼 있어 불필요한 객체 생성과 성능 저하를 유발할 수 있다는 점을 발견했습니다. 이 문제를 개선하고 싶다는 생각이 들었고, 직접 PR을 작성하기로 결정했습니다.

저는 해당 코드를 StringBuilder 기반으로 리팩터링하며 메모리 할당과 문자열 연산 비용을 줄였고, 내부 중복 코드까지 정리해 유지보수성을 높였습니다. 오픈소스 특성상 단일 환경에서 명확한 수치를 측정하기는 어렵지만, Spring Kafka는 국내외 여러 대규모 서비스에서 공통적으로 사용되는 핵심 라이브러리이기 때문에 이러한 미세한 연산 비용 절감도 다양한 서비스 환경에서 누적되어 실제 성능 향상으로 이어질 수 있다는 점에서 더 의미 있는 개선이라고 판단했습니다.

이 과정은 단순히 코드를 고치는 수준이 아니었습니다. 리뷰어의 높은 기준을 충족하기 위해 근거 기반 설명이 필요했고, 여러 차례 커밋을 조정해야 했습니다. 특히 리뷰 도중 Java 버전 차이로 인해 리뷰어가 코드 동작을 혼동한 부분이 있었는데, 저는 해당 코드를 직접 컴파일하고 테스트하며 실제 동작을 검증했고, 그 결과를 근거로 정확한 기술 정보를 제공해 잘못된 이해를 바로잡았습니다. 이는 단순 코드 기여가 아니라, 기술적 오해를 해소하고 리뷰 과정 전체의 신뢰도를 높인 의미 있는 경험이었습니다.

리뷰어가 해외에 있어 새벽 시간에 리뷰 알림이 오기도 했지만 저는 알림이 뜨는 즉시 대응하며 의견을 조율했습니다. 문제 분석 → 개선 전략 설계 → 리팩터링 → 검증 → 리뷰 대응까지 전 과정을 7일이라는 짧은 기간 동안 몰입해 완주했다는 점 또한 제 실행력과 집중력을 보여주는 경험이었습니다. 이 과정에서 AI는 변경 범위 파악과 시나리오 검증을 빠르게 도와주며 제 몰입 속도를 크게 높여주는 파트너가 되어주었습니다.

결국 PR은 승인되어 Spring Kafka 메인 저장소에 머지되었습니다. 이 경험은 “작은 문제라도 직접 실행하면 실제 개선을 만들 수 있다”는 것을 체감하게 해주었고, 글로벌 오픈소스 프로젝트의 리뷰 기준이 얼마나 엄격하고 체계적인지 배우는 계기도 되었습니다. 무엇보다 한 가지 문제에 깊게 몰입하고, 기술적 오해를 바로잡고, 끝까지 해결해내려는 집요함이 제 강점이라는 것을 스스로 증명할 수 있는 경험이었습니다.

### 4. 우리 회사에 지원하려는 이유는 무엇인가요? (500자 이내)
"정산플랫폼팀이 매일 수백억 원의 데이터를 Spring Batch로 처리한다는 점에서 제 경험과 강한 시너지를 낼 수 있다고 확신합니다.

저는 Spring Batch를 활용하여 대용량 데이터 처리 중 발생하는 업데이트 누락 문제를 커스텀 PagingReader를 구현해 해결한 경험이 있으며, 이 과정에서 배치 성능 최적화의 중요성을 깊이 체감했습니다. 특히 Spring Batch 오픈소스에 직접 기여한 PR #4931에서 DefaultFieldSet의 indexOf() 성능을 O(n)에서 O(1)로 개선하여, CSV 파일 처리 성능을 향상시켰습니다. 이는 정산 데이터처럼 컬럼이 많은 대용량 파일 처리에 직접적인 도움이 되는 개선사항입니다.

정산플랫폼팀이 추구하는 ""복잡함을 단순함으로 바꾸는"" 철학은 제가 코드 레벨에서 추구해온 가치관과 일치합니다. 정산이라는 미션 크리티컬한 도메인에서 신뢰성 있는 시스템을 구축하고, 3000개 이상의 테스트 코드로 비즈니스를 뒷받침하는 문화 속에서 성장하고 싶습니다. 제 Spring Batch 최적화 경험과 문제 해결 능력으로 팀의 ""정산을 쉽게 만드는 플랫폼"" 미션에 기여하고 싶습니다."
